Лабораторна робота №2Тема: Розробка вимог, уточнення діаграми класів та реалізація зв’язків агрегації та композиції.1. Вимоги до системи (Requirements)Функціональні вимоги:Система має забезпечувати облік домашніх тварин із базовими атрибутами: ID, ім'я, вік та стан здоров'я.Програма повинна підтримувати спеціалізацію тварин, зокрема виокремлення собак із зазначенням їхньої породи.Необхідно реалізувати механізм перевірки стану здоров'я тварини з виведенням статусу в консоль.Система повинна ідентифікувати власників (Adopter) за їхнім ID та повним іменем.Зв'язки: Реалізувати можливість закріплення тварин за власником (агрегація) та наявність невід'ємного медичного паспорта для кожної тварини (композиція).Технічні вимоги:Використання мови Java з дотриманням принципів інкапсуляції та успадкування.Перевизначення методів toString(), equals() та hashCode() для коректної роботи з об'єктами.Забезпечення унікальності об'єктів за полем id.2. Уточнення діаграми класів (Class Diagram Logic)На основі наданого коду та нових завдань, структура системи виглядає наступним чином:Inheritance (Успадкування): Клас Dog є похідним від Pet.Aggregation (Агрегація): Клас Adopter містить список об'єктів Pet. Тварини можуть існувати окремо від власника (наприклад, у притулку), але власник може мати декількох улюбленців.Composition (Композиція): Клас Pet тепер містить об'єкт MedicalPassport. Паспорт створюється одночасно з твариною і не може існувати без неї.
